## 1. 版本摘要

v128 是 Mneme v4.1.1 的仿真硬化版本，核心目标：**修复全部仿真 FAIL → P0 缺口清零**。

| 指标 | v127 | v128 |
|------|------|------|
| P0 阻断 | 1 | **0** |
| P1 重要 | 2 | **1** |
| P2 建议 | 4 | **4** |
| 仿真 FAIL | 3 | **0** |
| 仿真通过率 | ~95% | **~98%** |

---

## 2. 新增能力

| 能力 | 说明 |
|------|------|
| L7 事件溯源 | `event_log` / `sync` / `graph_triggers` 路由 + DB 层 |
| 批量导入 by-asset | `POST /api/v4/import/by-asset` 批量创建处理任务 |
| 子库 CRUD 增强 | `PUT` 更新 + 策略检查 + 分页返回 |
| 启动种子管道 | `seed_default_asset_import_pipelines` + `bootstrap_sub_libraries` |
| 上下文组装 | `context_packs` 表自动部署，assembly engine 可访问 |

## 3. 问题修复（7 个 P0 清零 + 4 个仿真 FAIL 修复）

### 3.1 P0 阻断项修复

| # | 问题 | 域 | 修复方式 |
|---|------|-----|----------|
| P0-1 | Agent Token Schema 前后端不一致 → POST 422 | Agent | Schema 对齐，Revoke 方法修正 |
| P0-2 | Gateway Provider/Binding POST 全 503 → JSONB 类型绑定缺失 | API 管理 | 修复 JSONB 绑定 + 事务封装 |
| P0-3 | pipeline_registry DAL `text[]` 格式 bug → `json.dumps()` 写 PG 数组 | 知识库 L3 | 改用 `to_pg_array()` + SQL `CAST AS text[]` |
| P0-4 | deploy.sh 硬编码数据库密码 | 安全 | 改为环境变量引用 |
| P0-5 | Agent 缺少 model_id 字段 | Agent | ✅ 已修复 — migration 0023, schema, DAL 全部添加 model_id |
| P0-6 | Pipeline Worker 缺失 → processing_jobs 永久 queued | 知识库 L4 | Worker 集成 processing_jobs 消费 |
| P0-7 | agent_cards 迁移未部署 | 记忆库 | alembic upgrade head 纳入部署流程 |

### 3.2 仿真 FAIL 修复

| FAIL | 问题 | 根因 | 修复 |
|------|------|------|------|
| #1 Pipeline CRUD 503 | `pipeline_registry` 表 `text[]` 列，代码用 `json.dumps()` 产生 JSON 字符串 | PG 无法隐式转换 | 改用 `to_pg_array()` + SQL `CAST(:col AS text[])` |
| #2 上下文组装 503 | `context_packs` 表未在远程服务器部署 | alembic upgrade head 未执行 | 确保 `deploy.sh` 步骤 2 执行 `alembic upgrade head` |
| #3 AssetTab 缺少 project_id | `project_id` 为必填字段 | — | 改为可选 (`UUID \| None = None`) |
| #4 StoreTab 缺少 Idempotency-Key | API 契约要求所有写操作携带 `Idempotency-Key` Header | — | API 契约要求，前端已对接 |
